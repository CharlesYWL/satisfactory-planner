## 3. 配平算法核心

### 正向 (产线取向) — 新手友好,这是重点
输入: 目标产品 + 各原料供给速率(默认原矿,可改)
```
1. 选目标产品的配方(默认 base,可换替代)
2. 自顶向下递归展开配方树,记录每个中间产物的"每 1 成品需求量/min"
3. 找瓶颈: 用户给的原料供给 ÷ 该原料的单位需求 = 可支撑的成品产量上限
   取所有原料里最小的那个 = 实际成品产出
4. 反推每级: 机器数 = ceil(该级需求 / 单机产能)
   - 严格整数模式: 显示利用率(实际需求/机器满载)
   - 超频模式: 机器数 = ceil(需求/(单机产能×2.5)),再算每台超频%
5. 标注: 红=瓶颈/欠料, 绿=过剩, 黄=非整数欠频
```

### 反向 (成品取向) — 对标原网站
```
目标 X 个成品/min → 递归: 每级机器数 = 需求/min ÷ 单机产能/min
原料叶子节点汇总 = 需要多少原矿/min
(机器数允许小数,跟原网站一致;也可切"整数+超频"显示)
```

### 关键公式
- 单机产能/min = produce_qty × 60 / duration(× 超频倍率,1.0~2.5)
- 超频功耗 = 基础功耗 × 超频倍率^1.321321
- 带子等级: Mk1=60, Mk2=120, Mk3=270, Mk4=480, Mk5=780, Mk6=1200/min

## 4. UI 设计 (对标原网站 + 改进)

### 布局
- 左: 流程图画布 (React Flow)
- 右: 配置面板, 3 个 Tab —— 产出 / 原料 / 选项

### 流程图节点
- 圆形机器图标 + 名称 + **整数台数**(正向)或 x1.34(反向)
- 节点下: 超频% / 利用率
- 连线: 物品名 + 速率, 颜色随物品(数据里有 color 字段)
- 颜色圈: 红=原料输入, 绿=加工, 终点=成品大图
- 总功耗显示在角落

### 替代配方下拉 (你的核心痛点)
- **只显示与当前产线相关的配方**: 遍历当前产线涉及的所有中间产物,
  用 `producers[item]` 取出能生产它的配方,只把这些塞进 dropdown
- 不是全 game 几百个糊脸 ✅

### 原料输入
- 默认所有叶子=原矿
- 每个原料行: 下拉选带速(60/120/240/自定义数字) + 可勾"当作已有半成品"

### 配色
- 深色模式 (#222) + Satisfactory 橙 + 物品色连线
